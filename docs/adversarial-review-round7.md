## 挑刺方 🔴

### T1. 新领域处理会卡住复盘 🔴
v5 新领域流程每遇到一个缺失 skill 就要暂停等用户确认。对话涉及 3 个新领域 = 中断 3 次。

### T2. 首次使用冷启动，无可用的领域 skill 🟡
第一个 `/复盘` 所有领域都是新的，全靠 AAR 4 问通用框架，复盘深度显著下降。

### T3. 降级路径缺失 🟡
WebSearch 失败、对话过长、MEMORY.md 损坏等异常情况未设计处理方式。

### T4. 迭代机制"回到阶段1"表述有问题 🟢
"重新评估"暗示重新扫描整个对话，实际只需对新发现归类+领域判定。

### T5. "新信息量占比 <20%"是伪精确 🟢
Agent 无法可靠量化百分比，但作为定性指引有意义。

### T6. 设计复杂度可能超出 Agent 可靠执行范围 🔴
7 轮审查累积的复杂度（前置+4阶段+迭代+闭环+7种关系+启发式+确认边界）可能让 Agent 执行不可靠。

## 调研与设计

### 降级路径调研要点
- **MAST 研究 (NeurIPS 2025)**：41-86.7% 多步 Agent 任务没有错误处理会失败
- **每步可靠性复利效应**：99%/步 × 20 步 = 82% 整体
- **三级分类**：Transient（重试）/ Permanent（降级）/ Critical（通知）
- **Recovery Ladder**：重试 → 降级 → 通知
- **不静默原则**：所有失败在汇报中标注

### v6 新增降级表
覆盖 12 个具体失败场景，每个有：失败场景、类型、降级策略、汇报标注。

### 通用原则
遇到预设表未覆盖的新异常 → 分类 → Transient重试1次 → 搜索解决方案 → 无法解决则跳过 → 标注 → 继续。
